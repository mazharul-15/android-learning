# Android Studio

> Android Studio is the official Integrated Development Environment (IDE) for Android app development. It provides all the tools needed to design, develop, test, debug, and publish Android applications.

---

# 📖 Overview

Android Studio is built on **IntelliJ IDEA** and is developed by **Google** specifically for Android development. It combines a code editor, visual layout editor, emulator, debugger, profiler, and build tools into one application.

Whether you're building a simple "Hello World" app or a production-scale application, Android Studio is the primary tool you'll use throughout the development process.

---

# 🎯 Why Android Studio?

Android Studio simplifies Android development by providing:

- A powerful code editor with Kotlin and Java support
- Intelligent code completion and suggestions
- XML layout editor with drag-and-drop support
- Android Emulator for testing apps
- Gradle integration for project builds
- Debugging and profiling tools
- Version control integration (Git)
- APK and App Bundle generation

Without Android Studio, Android development would be much more time-consuming and difficult.

---

# 🏗️ Main Components of Android Studio

## 1. Code Editor

The Code Editor is where you write Kotlin, Java, XML, and Gradle files.

Features:

- Syntax highlighting
- Auto-completion
- Error detection
- Code formatting
- Refactoring tools
- Quick fixes

Example:

```kotlin
fun main() {
    println("Hello Android!")
}
```

---

## 2. Project Window

The Project window displays your project's files and folders.

Common folders include:

```
app/
│
├── manifests/
├── java/
├── kotlin/
├── res/
└── Gradle Scripts/
```

You'll learn these folders in detail in **Project-Structure.md**.

---

## 3. Toolbar

The toolbar contains shortcuts for common tasks such as:

- Run App ▶
- Debug App 🐞
- Device Selector
- Sync Project
- Build Project

---

## 4. Device Manager

Device Manager allows you to create and manage Android Virtual Devices (AVDs).

You can:

- Create emulators
- Delete emulators
- Configure virtual devices
- Start or stop emulators

---

## 5. Layout Editor

The Layout Editor is used to design user interfaces.

It supports:

- Drag and Drop Design
- XML Editing
- Preview
- Constraint Layout tools

Example XML:

```xml
<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Hello Android"/>
```

---

## 6. Logcat

Logcat displays logs generated by your application and the Android system.

Example:

```kotlin
Log.d("MainActivity", "Button Clicked")
```

Useful for:

- Debugging
- Viewing crashes
- Monitoring application behavior

---

## 7. Build Output

Shows:

- Build progress
- Compilation errors
- Warnings
- APK generation status

---

## 8. Profiler

Android Profiler helps monitor:

- CPU usage
- Memory usage
- Network activity
- Battery usage

Useful for performance optimization.

---

## 9. Terminal

Android Studio includes a built-in terminal.

Example commands:

```bash
git status
git add .
git commit -m "Initial commit"
```

---

## 10. Version Control

Android Studio integrates with Git.

Features:

- Commit changes
- Push to GitHub
- Pull updates
- Branch management
- Compare file changes

# ⚠️ Common Beginner Mistakes

### ❌ Deleting important project files

Always understand a file before deleting it.

---

### ❌ Editing generated files

Avoid modifying files inside:

```
build/
```

They are regenerated automatically.

---

### ❌ Ignoring Gradle Sync errors

Gradle errors can prevent your project from building. Fix them before continuing.

---

### ❌ Running on an unsupported emulator

Choose a device with a compatible Android version and enough RAM.
